# 安全审查

## 触发场景

- 用户请求安全审查或代码审计
- 修改认证/授权相关代码
- 处理用户输入或外部数据
- 涉及文件上传、支付、个人信息

## 审查清单

### 1. XSS（跨站脚本）

- [ ] 用户输入是否通过 `v-text` 而非 `v-html` 渲染
- [ ] 必须使用 `v-html` 时是否经过 DOMPurify 等消毒
- [ ] URL 参数是否直接插入 DOM

### 2. 敏感信息

- [ ] Token / API Key 是否硬编码在源码中
- [ ] 敏感配置（密钥、Token）是否放在 `.env.local` / `.env.*.local`（已 gitignore），而非提交到仓库的 `.env.*` 文件
- [ ] Console.log 是否输出敏感数据
- [ ] 错误信息是否暴露内部实现细节

### 3. 认证与授权

- [ ] Token 存储是否通过 `src/utils/auth.ts` 封装读写（key：`ACCESS_TOKEN`）
- [ ] 请求拦截器是否正确附加/刷新 Token
- [ ] 路由是否有权限守卫（`router.beforeEach`）
- [ ] API 权限是否在前后端双重校验

### 4. 请求安全

- [ ] 是否存在 CSRF 防护（SameSite Cookie / CSRF Token）
- [ ] 是否正确处理 CORS
- [ ] 上传文件是否校验类型和大小
- [ ] 是否存在 SSRF 风险（用户可控 URL 请求）

### 5. 依赖安全

- [ ] 是否使用了有已知漏洞的依赖版本
- [ ] 第三方脚本是否来自可信 CDN

## 输出格式

```
## 安全审查报告

### 🔴 高危
- [文件:行号] 问题描述 → 修复建议

### 🟡 中危
- [文件:行号] 问题描述 → 修复建议

### 🟢 低危 / 建议
- [文件:行号] 问题描述 → 修复建议

### ✅ 通过项
- 列出已检查且未发现问题的方面
```

## 修复优先级

1. 高危 — 立即修复，阻塞合并
2. 中危 — 当前迭代内修复
3. 低危 — 排入后续优化
